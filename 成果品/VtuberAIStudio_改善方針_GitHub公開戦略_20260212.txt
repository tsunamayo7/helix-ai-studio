■ アプリケーション概要
  名称: VTuber Personality Analyzer (VtuberAIStudio)
  バージョン: v3.6.11
  技術スタック: Python 3.10+ / PyQt6 / SQLAlchemy / SQLite
  用途: VTuber配信アーカイブの収集・文字起こし・性格プロファイリング

■ 主要機能
  1. YouTube配信アーカイブの自動ダウンロード (yt-dlp)
  2. 音声文字起こし (faster-whisper / GPU・CPU対応)
  3. 動画フレーム解析 (Gemini 2.5 Flash / Claude Vision)
  4. ライブチャット・コメント収集
  5. X(Twitter)投稿収集 (Playwright)
  6. Big Five性格特性プロファイリング (Claude Opus/Sonnet)
  7. 全文検索 (SQLite FTS5)
  8. マルチVTuber対応 (SQLAlchemy マルチテナント)
  9. RAG拡張 (エンベディングキャッシュ)

■ アーキテクチャ構成
  src/collectors/   - データ収集層 (YouTube, X, Web, Holodex)
  src/analyzers/    - 解析層 (音声認識, 動画解析, テキスト強化)
  src/database/     - データベース層 (20+テーブル, SQLAlchemy ORM)
  src/personality/  - AI処理層 (Big Fiveプロファイリング)
  src/gui/          - UI層 (PyQt6, タブ構成, バックグラウンドワーカー)
  src/api/          - API層 (Claude Code CLI / APIキーデュアルモード)
  src/utils/        - ユーティリティ (設定管理, 暗号化, ログ)

■ 特筆すべき設計思想
  - 「No-API Data Collection」: YouTube/Google APIなしでyt-dlp+ブラウザ自動化で動作
  - ヒューマナイズドリクエスト: 5-10秒遅延 + 10アイテムごとのコーヒーブレイク
  - Fernet暗号化によるAPIキー保護
  - Claude Code CLI (Maxプラン OAuth) / APIキーのデュアルモード

■ コード規模
  ソースコード: 約30ファイル (src/ 以下)
  データベース: 20+テーブル (SQLAlchemy ORM)
  EXE: 2.8GB (ffmpeg/ffprobe同梱)

========================================
A. コード品質・設計の改善
========================================

A-1. セキュリティ強化
  現状: data/.key (暗号化キー) がリポジトリに含まれるリスク
  対策:
    - .gitignore に data/.key, config.yaml, *.db を確実に追加
    - APIキーをハードコードしている箇所がないか全ファイル監査
    - .env ファイルによる環境変数管理への移行を検討
    - ffmpeg.exe/ffprobe.exe (各195MB) はリポジトリから除外し、
      初回起動時の自動ダウンロードまたはREADMEでの手動配置手順に変更

A-2. EXEサイズの最適化
  現状: 2.8GB (ffmpeg/ffprobe同梱)
  対策:
    - ffmpeg/ffprobe をバンドルから分離 (ユーザーが別途インストール)
    - PyInstaller の --exclude-module で不要モジュール除外
    - GitHub Releases での配布 (リポジトリ本体には含めない)

A-3. バージョン管理の統一
  現状: main.py (v3.6.11) と constants.py (v3.6.10) でバージョン不一致
  対策:
    - src/utils/constants.py を Single Source of Truth にする
    - または pyproject.toml / setup.cfg でバージョンを一元管理

A-4. コードスタイル統一
  対策:
    - black フォーマッターの導入 (pyproject.toml に設定)
    - flake8 / ruff による静的解析
    - pre-commit フックの設定

A-5. テストの充実
  現状: tests/ ディレクトリは存在するが、実行可能性・カバレッジ不明
  対策:
    - CI で pytest を自動実行
    - 主要モジュール (collectors, analyzers, database) のユニットテスト整備
    - モック活用でAPI依存テストを分離

A-6. 依存関係の整理
  対策:
    - requirements.txt にバージョン固定 (==) を明記
    - requirements-dev.txt を分離 (pytest, black, flake8 等)
    - pip-audit による脆弱性チェック

A-7. エラーハンドリングの改善
  対策:
    - グローバル例外ハンドラーの導入 (未処理例外のキャッチ)
    - crash.log への自動書き出し
    - ユーザーフレンドリーなエラーダイアログ

第5部: VtuberAIStudio の強みと差別化ポイント
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

GitHub公開において、以下の独自性がプロジェクトの訴求力となる。

  1. No-API アプローチ
     YouTube Data API v3 のクォータ制限を回避し、yt-dlp ベースで
     データ収集を実現。APIキー不要で即座に利用開始可能。

  2. VTuber特化の性格分析
     Big Five性格特性モデルを VTuber コンテンツに適用するという
     ニッチだが需要のあるユースケース。類似OSSは極めて少ない。

  3. マルチモーダル解析パイプライン
     音声(Whisper) + 映像(Gemini/Claude) + テキスト(NLP) の
     統合分析は技術的にアドバンスな構成。

  4. コスト最適化設計
     Gemini 2.5 Flash (Claude比 1/40のコスト) をフレーム解析に、
     Claude をプロファイリングに使い分けるハイブリッド戦略。

  5. ヒューマナイズドスクレイピング
     遅延とブレイクを組み込んだ「人間らしい」リクエストパターンは
     倫理的スクレイピングの実装例として参考になる。

  6. SQLite FTS5 全文検索
     ローカル完結の高速全文検索は、プライバシー重視の
     ユーザーに訴求力がある。

第6部: リスク・注意事項
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  1. 法的リスク
     - YouTube利用規約: yt-dlp による動画ダウンロードは利用規約に
       抵触する可能性がある。READMEに「研究目的・個人利用」の
       免責事項を明記する必要がある。
     - X(Twitter) スクレイピング: 規約違反の可能性。
       Playwright による自動化は明確にグレーゾーン。
     - 推奨: 「本ツールは教育・研究目的で提供されています」の
       ディスクレイマーを LICENSE / README に記載。

  2. プライバシーリスク
     - VTuberの個人情報推定につながる解析結果の公開禁止を明記
     - プロファイルデータの取り扱い指針の策定
     - data/ ディレクトリの .gitignore 確認

  3. API依存リスク
     - Claude Code CLI の仕様変更への追従
     - Gemini API の価格変更・廃止リスク
     - yt-dlp の YouTube 仕様追従

  4. 大容量ファイル
     - ffmpeg.exe (195MB), ffprobe.exe (195MB), EXE (2.8GB) は
       Git リポジトリに含めない
     - Git LFS の利用、または GitHub Releases での配布
