## 0. はじめに — 初回起動ガイド

### 前提条件

| 項目 | 要件 |
|------|------|
| OS | Windows 10/11 または macOS 12+ |
| Python | 3.10 以上（3.11 推奨） |
| RAM | 16 GB 以上 |
| GPU | NVIDIA + CUDA（ローカルLLM用、任意）。macOS は Metal/CPU |

### 起動方法

```bash
git clone https://github.com/tsunamayo7/helix-ai-studio.git
cd helix-ai-studio
pip install -r requirements.txt
python HelixAIStudio.py          # macOS: python3
```

### 初回にやること

1. **ローカルLLMを使う場合**: [Ollama](https://ollama.com) をインストール → **Ollama設定タブ** で接続テストを実行
2. **クラウドAIを使う場合**: **一般設定タブ → APIキー管理** で使用するプロバイダのキーを入力・保存
3. **両方を連携させる場合（mixAI）**: 上記1+2を両方設定した後、**mixAI タブ → 設定** でエンジンとモデルを選択

## 1. 共通操作

### 1.1 メインタブバー（画面最上部）

画面最上部に8つのメインタブが並んでいます。クリックで各機能に切り替えます。

| タブ名 | 概要 |
|---|---|
| **mixAI** | 3+1 Phase AIパイプライン（クラウド+ローカルLLM連携） |
| **soloAI** | 統合AIチャット — Cloud AI（Claude/GPT/Gemini）+ Ollama を一つのタブで。旧 cloudAI + localAI の後継 |
| **CloudAI設定** | クラウドモデル管理・認証方式・実行オプション・MCP設定 |
| **Ollama設定** | Ollama接続・モデル管理・ローカルLLM関連設定 |
| **History** | 統合チャット履歴（旧 cloudAI/localAI ログも対応） |
| **RAG** | ドキュメントからの知識ベース構築・質問 |
| **Virtual Desktop** | 隔離された仮想デスクトップ（Windows Sandbox 標準 / Docker 任意） |
| **一般設定** | APIキー・表示設定・Web UI等のアプリ全体設定 |

### 1.2 言語切り替え（画面右上）

- **「日本語」ボタン**: アクティブ時はシアン色。日本語UIに切り替え
- **「English」ボタン**: 英語UIに切り替え

切り替えはリアルタイムで全タブに反映されます。

### 1.3 ステータスバー（画面最下部）

- **左端**: 現在の処理状態（`Ready`、実行中のフェーズ名など）
- **右端**: バージョン表示（例: `v12.8.0`）

### 1.4 会話継続パネル（共通コンポーネント）

mixAI / cloudAI / localAI の3タブに共通で、チャット画面の右下に表示されます。
AIが質問や確認を返してきた際に、素早く応答するための専用パネルです。

| 要素 | 操作 |
|---|---|
| **継続入力欄** | 追加の指示やメッセージを入力 |
| **「はい」ボタン**（緑） | 「はい」を即座に送信 |
| **「続行」ボタン**（青緑） | 「続行」を即座に送信 |
| **「実行」ボタン**（紫） | 「実行」を即座に送信 |
| **「→ 送信」/「→ 継続送信」ボタン** | 入力欄のテキストで処理を継続 |

### 1.5 Helix Pilot スクリーンショットボタン（右下角）

全タブ共通で画面右下角に表示される丸ボタンです。クリックすると現在のウィンドウのスクリーンショットを撮影します（Helix Pilot GUI自動操作ツール連携用）。

## 2. mixAI タブ — 統合オーケストレーション

5つのフェーズで複数のAIモデルを連携させるパイプラインです。
クラウドAI（Claude / GPT / Gemini）が計画と検証を担当し、ローカルLLM（Ollama）が実行を担当することで、コストを抑えながら高品質な結果を得られます。

### 2.1 サブタブ

- **チャット**: メインの実行・表示画面
- **設定**: パイプラインの詳細設定

---

### 2.2 チャット サブタブ

#### ヘッダー領域

| 要素 | 説明 |
|---|---|
| **タブタイトル** | 「mixAI - 統合オーケストレーション」 |
| **ステータスメッセージ** | 「準備完了。メッセージを入力してください。」等の現在状態 |

#### チャット表示領域（中央大画面）

- AIの応答がHTML形式で表示されます
- 各フェーズ（Phase 1〜4）の進行状況がリアルタイムで表示
- コードブロックはシンタックスハイライト付き
- ユーザーメッセージは右寄せ、AIメッセージは左寄せ

#### ツール実行ログ（折りたたみ式）

| 要素 | 説明 |
|---|---|
| **「ツール実行ログ (クリックで展開)」** | クリックで展開/折りたたみ。Claude CLIのツール使用ログを表示 |

#### メッセージ入力エリア（画面下部）

| 要素 | 操作 |
|---|---|
| **テキスト入力欄** | プロンプトを入力。`Ctrl+Enter` で送信 |
| **添付ボタン** | ファイルを添付（画像、テキスト等） |
| **スニペットボタン** | 定型プロンプト（スニペット）メニューを表示。VDコーディング用テンプレート等 |
| **キャンセルボタン** | 実行中の処理を中断 |
| **実行ボタン** | 3+1 Phase パイプラインを開始 |

---

## 5. History タブ — チャット履歴

### 5.1 フィルタバー（上部）

| 要素 | 操作 |

  - **モデル名**

## 9. よくある操作フロー

### 9.1 ローカルLLMとチャットしたい

1. [Ollama](https://ollama.com) をインストール・起動
2. **localAI タブ → 設定** → ホストURLを確認 → 「接続テスト」
3. モデルがなければ、モデル名入力欄に `gemma3:4b` と入力 → 「プル」
4. **localAI タブ → チャット** → モデル選択コンボでモデルを選ぶ → メッセージ入力 → 送信

### 9.2 Claude / GPT / Gemini とチャットしたい

1. **一般設定タブ → APIキー管理** で使うプロバイダのキーを入力・保存
2. **cloudAI タブ → チャット** → 上部の「…」ボタンでモデルを選択 → メッセージ入力 → 送信

### 9.3 mixAIパイプラインを実行したい

1. クラウドAIのAPIキーを設定済みであること（Phase 1/3 用）
2. Ollamaにモデルがインストール済みであること（Phase 2 用）
3. **mixAI タブ → 設定** で各フェーズのエンジンとモデルを設定 → 「全設定を保存」
4. **mixAI タブ → チャット** → プロンプト入力 → 「実行」

### 9.4 RAG知識ベースを構築して質問したい

1. **RAG タブ → 構築** → 「フォルダを開く」で対象ドキュメントを確認
2. **RAG タブ → 設定** で使用モデルを選択
3. **RAG タブ → 構築** → 「プラン作成」→ 「ビルド開始」
4. 構築完了後、**RAG タブ → チャット** で質問

### 9.5 Docker仮想デスクトップを使いたい

1. Docker Desktop をインストール・起動
2. **Virtual Desktop タブ → 設定** → Docker ステータスが「起動中」であることを確認
3. イメージが未ビルドなら「イメージビルド」を実行
4. **Virtual Desktop タブ → デスクトップ** → 「起動」ボタン
5. VNCビューア内で操作。完了したら「差分確認」→ ファイルを選択 → 「適用」でホストに反映

## 10. トラブルシューティング

| 症状 | 対処 |
|---|---|
| **Ollamaに接続できない** | Ollamaが起動しているか確認 → localAI タブ → 設定 → 接続テスト。ホストURLが `http://localhost:11434` であることを確認 |
| **クラウドAIがエラーを返す** | 一般設定 → APIキー管理でキーが正しく入力されているか確認。APIの利用制限に達していないか確認 |
| **mixAIのPhase 2が失敗する** | Ollamaに必要なモデルがインストールされているか確認。mixAI → 設定 → 「Ollama接続テスト」を実行 |
| **VNCが表示されない** | Docker Desktopが起動しているか確認。VD → 設定 → Dockerステータスを確認。「ブラウザで開く」で外部ブラウザを試す |
| **日本語が文字化けする** | Python環境のエンコーディングがUTF-8であることを確認。`set PYTHONUTF8=1` を設定して再起動 |
| **送信ボタンが反応しない** | 入力欄が空でないか確認。前の処理が実行中の場合は完了を待つかキャンセル |
